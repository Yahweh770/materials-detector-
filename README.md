# Учёт материалов — Единая база

Приложение для учёта материалов с отслеживанием сроков годности и просроченных документов.

## Требования

- **Python 3.8 или выше** (обязательно добавить в PATH при установке)
- Windows 10/11

## Быстрый старт

### Вариант 1: Автоматический запуск (рекомендуется)

Просто запустите файл `run.bat` двойным кликом. Скрипт автоматически:
1. Проверит наличие Python
2. Создаст виртуальное окружение
3. Установит все необходимые зависимости
4. Запустит приложение

### Вариант 2: Ручная установка

```bash
# Установка зависимостей
pip install -r requirements.txt

# Запуск приложения
python materials.py
```

## Функционал

### Основные возможности:
- ✅ Добавление, редактирование и удаление записей о материалах
- ✅ Поиск по всем полям записи
- ✅ Экспорт данных в Excel с форматированием
- ✅ Работа с несколькими файлами баз данных
- ✅ Копирование данных в буфер обмена

### Отслеживание сроков:
- 📊 **Статус материалов** — окно с полной информацией по всем материалам:
  - Просроченные/истекающие документы
  - Действительные документы
  - Материалы без срока хранения
  
- ⚠️ **Просроченные документы** — отдельное окно только с проблемными документами

- 🔔 **Индикатор в нижней панели** — показывает актуальную информацию:
  - Количество просроченных документов
  - Количество документов, истекающих в течение 30 дней
  - Кликабелен — открывает окно статуса материалов

## Как увидеть окно со статусом материалов

Есть несколько способов открыть окно с актуальной информацией о сроках:

1. **Через меню "Сервис"** → "📊 Статус материалов"
2. **Кнопка на панели инструментов** → "⏰ Срок годности"
3. **Клик по индикатору** в нижней панели (где написано "🔴 Просрочено: X" или "✅ Все документы действительны")
4. **Через меню "Сервис"** → "⚠️ Просроченные документы" (только проблемные записи)

## Решение проблем

### Python не найден
1. Скачайте Python с https://www.python.org/downloads/
2. При установке **обязательно отметьте галочку** "Add Python to PATH"
3. Перезапустите командную строку и попробуйте снова

### Ошибки при установке зависимостей
Запустите от имени администратора:
```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### Приложение не запускается
Удалите папку `venv` (если существует) и запустите `run.bat` заново.

## Поддержка

При возникновении проблем проверьте:
1. Версию Python (`python --version`)
2. Наличие всех зависимостей (`pip list`)
3. Логи ошибок в консоли
